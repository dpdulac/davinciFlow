The script must query ShotGrid for the latest (or historical) media files for a specific sequence of shots or a direct Playlist, download missing proxies, and assemble them chronologically into a DaVinci Resolve timeline using the `DaVinciResolveScript` and `fusion.UIManager` APIs.

Build a GUI window (Geometry: `[400, 400, 550, 450]`) using `fusion.UIManager`.
Organize it into distinct expandable/collapsible visual sections (`ui.VGroup`) toggled by Header Buttons (`▶ FLOW`, `▶ SHOT`, etc). Use a `create_toggle_handler` function to hide/show the VGroups.
The `AdvancedGrp` must be forced to `.Hide()` state upon initialization!

## 3. PATH RESOLUTION, PROXIES & LUTS
Create a `resolve_path(raw_path)` function:
- Replaces `/` with `\`.
- If path starts with `\s\prodanim\`, strip it and try checking the `W:\` drive, then fallback to `V:\`.
- If it starts with `V:\` but doesn't exist, smartly swap to `W:\` and check again (and vice versa).

Create an `ensure_luts_installed()` function:
- Checks `T:\davinciFlow_repo\luts`. Uses `shutil.copy2` to copy `.cube` files to DaVinci's system LUT folder at `C:\ProgramData\Blackmagic Design\DaVinci Resolve\Support\LUT\davinciFlow`.

## 4. SHOTGRID QUERY LOGIC (`fetch_flow_data`)
When "Build Sequence" is clicked, check the active Mode:

**If Sequence Mode:**
1. Find all `Shot` entities linked to `SeqCombo`.
2. Filter shots based on `ShotFilterLine` and `ExcludeCheck` using Regex to 4-pad leading digits (`re.match(r'^(\d+)(.*)$', p)`).
3. Query `Version` entities for all shots, filtered by `valid_tasks`.
4. Group versions by base name, prioritize `dnxhd` over `mjpeg`, sort by `created_at`.
5. Apply the `TakeCountCombo` to keep historical takes.

**If Playlist Mode:**
1. Bypass the Task hierarchy entirely.
2. Query Flow for the exact `Playlist` named in `PlaylistCombo`.
3. Extract the `versions` array from the playlist entity.
4. Fetch those exact versions by ID.
5. Extract the parent `Shot` entities from those versions to get timeline cutting (`sg_cut_in`, `sg_cut_out`).
6. Apply `ShotFilterLine` logic (e.g. if the user only wants shot 0050 out of the 100-shot playlist).
7. Force the version dictionary to only have one element (no historical takes).

## 5. MEDIA IMPORT & DAVINCI ASSEMBLY
1. Pre-flight phase: Gather all web proxy downloads where local file doesn't exist. Execute in parallel using `concurrent.futures.ThreadPoolExecutor(max_workers=10)`.
2. **Timeline Logic**:
   - In Sequence Mode: Base name is `Seq_YYYY_MM_DD`. If `UseLatestTimeline` is false, it appends a suffix index. If true, it clears the latest existing one.
   - In Playlist Mode: Base name is `{PlaylistName}_vXXX`. If `UseLatestTimeline` is false, it parses the timeline names via regex (`_v(\d{3})$`) to mathematically increment the padding (e.g. `_v001` becomes `_v002`). If true, it clears the clips in the highest version found.
3. Import media and takes to Media Pool.
4. Construct `clip_info` dicts (`{"mediaPoolItem": item, "is_hero": bool}`) for `AppendToTimeline()`. Calculate duration: `endFrame = sg_cut_out - sg_cut_in`.
5. **LUT Application:** Iterate through `appended_items`.
   - If EXR and is_hero: Apply standard `exr_lut`.
   - If EXR and NOT is_hero: Apply generated B&W LUT (`TMNT2_acescg_to_vd16_bw.cube`).
   - If Proxy and NOT is_hero: Apply generic `proxy_bw.cube`.
   *(LUTs are applied via `timeline_item.SetLUT(1, lut_path)`).*

## 6. MATHEMATICAL LUT GENERATOR (`generate_luts.py`)
Provide a standalone script (`generate_luts.py`) that uses raw math to bake Black & White transforms.
- For generic `proxy_bw.cube`: Generates a 33x33x33 identity cube, forcing `L = R*0.2126 + G*0.7152 + B*0.0722`.
- For the EXR LUT: Ingests the studio's EXR LUT, computes Rec.709 Luma `L`, and writes a new `.cube` where `R = G = B = L`.
