# DaVinci Flow Pipeline Script - AI Recreation Prompt

**Purpose:** This document is designed to act as a highly detailed prompt. If you feed this text to an LLM, it should be able to perfectly recreate the `davinciFlow.py` script from scratch.

## CONTEXT & GOAL
Write a Python 3 script named `davinciFlow.py` that bridges ShotGrid (Flow) and DaVinci Resolve. The script must query ShotGrid for the latest (or historical) media files for a specific sequence of shots, download missing proxies, and assemble them chronologically into a DaVinci Resolve timeline using the `DaVinciResolveScript` and `fusion.UIManager` APIs.

## 1. CONFIGURATION & AUTHENTICATION
The script must authenticate with ShotGrid via `shotgun_api3`.
It should fall back to default hardcoded credentials, but primarily attempt to load them from `davinciFlow_config.json`:
- `flow_url` (default: "https://mikrosanim.priv.shotgunstudio.com/")
- `script_name` (default: "resolveTest")
- `script_key` (default: "vblervsd(ubdZzxsxtvo5jimv")
- `proxy_download_path` (default: "T:\flowDavinci")
- `projects` (default: ["Tmnt2"])
- `tasks` (default: `["delivery", "confo_render", "compo_comp", "compo_precomp", "light_precomp", "anim_main", "layout_base", "previz_base", "editing_edt"]`)

It must also load a `userpref.json` (if it exists) which contains custom task arrays:
```json
{"presets": {"Compositor Default": ["compo_comp", "light_precomp", "anim_main", "editing_edt"]}}
```

## 2. UI DESIGN (Fusion UIManager)
Build a GUI window (Geometry: `[400, 400, 550, 450]`) using `fusion.UIManager`.
Organize it into three distinct visual sections using centered Labels (e.g. `--- FLOW ---`).

**Section: FLOW**
- `ProjectCombo`: Dropdown of projects from config.
- `SeqCombo`: Dropdown of sequence numbers (e.g. "0575") fetched live from SG upon launch.
- `ImageSeqCheck`: Checkbox (Use Image Sequences instead of movies).
- `UseAudio`: Checkbox (Import .wav files from Flow).

**Section: TASKS**
- `UsePresetCheck`: Checkbox to override High/Low drop-downs.
- `HighestTaskCombo` & `LowestTaskCombo`: Dropdowns side-by-side in an HGroup, populated from the master tasks list.
- `TaskPresetCombo`: Dropdown populated from `userpref.json` keys.
*(Dynamic UI Rule: If `UsePresetCheck` is checked, disable Highest/Lowest combos and enable TaskPresetCombo. Vice versa if unchecked. Attach this to a UI Clicked event).*

**Section: TIMELINE**
- `TakeCountCombo`: Dropdown ("None (Latest Only)", "Last 2 Versions", "Last 3 Versions", "Last 4 Versions", "Last 5 Versions", "All Versions").
- `UseLatestTimeline`: Checkbox to update the latest matching timeline (clears clips first).

**Footer**
- `CancelBtn` & `BuildBtn` (Build Sequence).

## 3. PATH RESOLUTION & PROXIES
Create a `resolve_path(raw_path)` function:
- Replaces `/` with `\`.
- If path starts with `\s\prodanim\`, strip it and try checking the `W:\` drive, then fallback to `V:\`.
- If it starts with `V:\` but doesn't exist, smartly swap to `W:\` and check again (and vice versa).

Create a `get_missing_media_path()` function:
- Downloads a red "NO CLIP" placeholder image from `https://dummyimage.com/1920x1080/ff0000/ffffff.jpg?text=NO+CLIP` to the proxy directory using `urllib.request`.

## 4. SHOTGRID QUERY LOGIC (`fetch_flow_data`)
When the user clicks "Build Sequence", query SG:
1. Find the Project ID.
2. Find all `Shot` entities where `sg_sequence` matches `SeqCombo`. Retrieve `code`, `sg_cut_in`, `sg_cut_out`, `sg_head_in`.
3. If `UseAudio` is checked, find `PublishedFile` entities of type `EditingSound` or `Sound` linked to the sequence. Resolve their paths.
4. Determine the `valid_tasks` array: If `UsePresetCheck` is checked, use the array from `userpref.json`. Otherwise, slice the `tasks` array from config using the Highest to Lowest indices.
5. Query `Version` entities for all found shots filtered by `valid_tasks`. Get `sg_path_to_movie`, `sg_path_to_frames`, `created_at`, `sg_uploaded_movie_mp4`.

**Version Filtering & Grouping:**
- For each shot, loop through `valid_tasks` in exact order. Stop at the first task that returns media.
- For that task, group versions by their base name (stripping out `-mjpeg`, `-dnxhd`, `.mov`).
- If duplicates exist for a base version, prioritize `dnxhd` (score=2) over `mjpeg` (score=1).
- Sort the unique versions by `created_at` (Newest first).
- Slice the array based on the `TakeCountCombo` limit (e.g. Last 2 Versions = slice `[:2]`).
- The newest version `[0]` is the base path. All others `[1:]` are stored in a `takes` list for that shot.
- If a version has no local path but has a web url, flag `is_web_proxy = True`.

## 5. MEDIA IMPORT & DAVINCI ASSEMBLY
1. Check if the timeline exists. If `UseLatestTimeline` is checked, clear all clips via `target_timeline.DeleteClips()`. Otherwise, create a new timeline `SeqNumber_YYYY_MM_DD`.
2. Sort the processed shots alphabetically by `shot_code`.
3. Loop through shots:
   - If `is_web_proxy` is true (for base or takes), use `urllib.request.urlretrieve` to download it to `PROXY_DOWNLOAD_PATH` before importing.
   - Import base media to `movies` bin in the Media Pool. Import take media to the same bin.
   - Construct `clip_info` dictionary for `AppendToTimeline()`. Calculate duration: `endFrame = sg_cut_out - sg_cut_in`. (If missing placeholder, default to 48 frames or the cut duration).
   - Track mapped Takes: Store the target `video_index` (len of current video clips list) alongside the imported take `MediaPoolItem`s.
   - Import Audio to `audio` bin if enabled. Map its `trackIndex` to 1.
4. Call `appended_items = media_pool.AppendToTimeline(video_clip_infos)`.
5. Iterate through `appended_items`. If an item's index matches our tracked Takes, call `timeline_item.AddTake(take_media_pool_item, startFrame=0, endFrame=duration)`.
6. Append Audio infos to timeline using `recordFrame = video_item.GetStart()` to perfectly sync.

## 6. EXECUTION
Show the window and start the Fusion `dispatcher.RunLoop()`. Hide window on completion or cancel.

## 7. COMPANION APP (`userpref_manager.py`)
Also write a secondary standalone script using `PySide6`. This is a dedicated Desktop app so artists don't have to manually edit `userpref.json`.

- It provides a `QLineEdit` for a preset name.
- It provides a `QComboBox` populated with the tasks, and an "Add Task" `QPushButton`.

- It has a "Save Preset" `QPushButton` that writes to `userpref.json`.
- It has a `QTableWidget` to view all saved presets in `userpref.json`.
